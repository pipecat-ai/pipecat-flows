- ⚠️ `FlowArgs` has been widened from `dict[str, Any]` to `Mapping[str, Any]` to reflect that handlers consume args as read-only input and to align with Pipecat's typing of `FunctionCallParams.arguments`. Runtime behavior is unchanged; the concrete object is still a `dict`. If you explicitly annotated a handler parameter with `FlowArgs` and then mutated it, the mutation will now surface as a type error. Stop mutating `args`, or annotate the parameter with `dict[str, Any]` if you need the write interface.
